# Sistema-de-horas-extras-front-end

Sistema de Horas Extras (Front-end)

E aí! Este projeto foi desenvolvido para um desafio da faculdade com o objetivo de resolver um problema real: o controlo de horas extras numa empresa. Criámos interfaces personalizadas para cada cargo, garantindo que técnicos, encarregados e gestores tenham as ferramentas certas para o seu dia a dia.
O que foi feito?
Até agora, o foco total foi no **Front-end**. Criámos toda a experiência visual, a lógica de navegação entre perfis e o design responsivo. O banco de dados já está estruturado, mas a integração direta com o site ainda será feita numa próxima etapa.

Como testar o sistema? (Logins de Teste)
Como ainda não ligámos o banco de dados, podes usar estes acessos de simulação diretamente na tela de login:

* Para entrar como Técnico:
    * Utilizador (Cadastro): tecnico
    * Palavra-passe: 123

* Para entrar como Encarregado:
    * Utilizador (Cadastro): encarregado
    * Palavra-passe: 456

* Para entrar como Gestor:
    * Utilizador (Cadastro): gestor
    * Palavra-passe: 789

O que cada perfil faz?

Perfil Técnico
O foco aqui é a execução. O técnico pode fazer o seu check-in, consultar a lista de tarefas e acompanhar se as suas solicitações de horas extras foram aprovadas ou reprovadas.

Perfil Encarregado
Este perfil funciona como uma ponte. Além de gerir o seu perfil e bater o ponto, o encarregado é quem cria as solicitações de horas extras para os técnicos da sua equipa.

Perfil Gestor
É quem tem a visão estratégica. O gestor pode aprovar ou rejeitar solicitações pendentes, publicar notícias internas para toda a empresa e analisar relatórios de custos e desempenho por setor.



Nota: Este é um projeto académico. Fica à vontade para explorar o código e as interfaces!
